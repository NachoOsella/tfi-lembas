## Cobertura por prioridad

| Prioridad | Cobertura esperada |
|---|---|
| Obligatorio | Debe tener caso de uso, regla y prueba critica. |
| Diferencial | Debe tener regla, caso de uso y al menos prueba funcional/integracion. |
| Futuro cercano | Puede quedar diseñado y documentado, sin implementacion completa inicial. |
| Restriccion | Debe aparecer en alcance/fuera de alcance y validarse en defensa. |
