# Requisitos funcionales

> [!info]
> El detalle de reglas asociadas esta en [[04.1 - Reglas de negocio]]. La relacion con casos de uso y pruebas esta en [[03.3 - Matriz de trazabilidad]].

## Seguridad, usuarios y roles

| ID | Requisito | Prioridad |
|---|---|---|
| RF-01 | El sistema debe permitir iniciar y cerrar sesion. | Obligatorio |
| RF-02 | El sistema debe administrar usuarios con alta, edicion, consulta y baja logica. | Obligatorio |
| RF-03 | El sistema debe diferenciar roles Administradora/Dueña y Empleado. | Obligatorio |
| RF-04 | El sistema debe restringir funcionalidades segun rol y permisos. | Obligatorio |
| RF-05 | El sistema debe impedir operar a usuarios inactivos. | Obligatorio |

## Compras simples a proveedor

| ID | Requisito | Prioridad |
|---|---|---|
| RF-78 | El sistema debe permitir registrar una compra simple asociada a un proveedor. | Diferencial |
| RF-79 | El sistema debe permitir precargar productos esperados con cantidad esperada y costo unitario esperado. | Diferencial |
| RF-80 | El sistema debe permitir guardar una compra en estado borrador sin impactar stock. | Diferencial |
| RF-81 | El sistema debe permitir controlar la recepcion cargando cantidad recibida real y costo unitario recibido. | Diferencial |
| RF-82 | El sistema debe permitir cargar fecha de vencimiento y lote de forma manual y opcional por cada producto recibido. | Diferencial |
| RF-83 | El sistema debe permitir dividir una linea recibida en varios lotes cuando el mismo producto llegue con distintos vencimientos. | Diferencial |
| RF-84 | El sistema debe advertir si un producto controla vencimiento y se confirma sin fecha. | Diferencial |
| RF-85 | Al confirmar la recepcion, el sistema debe generar movimientos de stock `INGRESO_COMPRA` usando la cantidad recibida real. | Diferencial |
| RF-86 | El sistema debe registrar diferencias entre cantidad/costo esperado y cantidad/costo recibido. | Diferencial |
| RF-87 | El sistema debe permitir agregar durante la recepcion un producto recibido que no estaba previsto. | Diferencial |
| RF-88 | El sistema debe permitir consultar compras por proveedor, fecha, producto o estado. | Diferencial |
| RF-89 | El sistema debe auditar confirmacion y anulacion de compras. | Diferencial |
| RF-90 | El sistema no debe gestionar facturas fiscales, impuestos, pagos, deuda ni cuenta corriente de proveedores dentro del modulo de compras simple. | Restriccion |
