# Estados y ciclos de vida

## Venta

```mermaid
stateDiagram-v2
    [*] --> BORRADOR
    BORRADOR --> CONFIRMADA: confirmar / stock + caja
    BORRADOR --> ANULADA: cancelar borrador
    CONFIRMADA --> ANULADA: anulacion controlada
    ANULADA --> [*]
```

Reglas: una venta solo impacta stock y caja al confirmarse. La anulacion no elimina la venta.

## Lista de precios

```mermaid
stateDiagram-v2
    [*] --> CARGADA
    CARGADA --> PROCESADA
    PROCESADA --> EN_REVISION
    EN_REVISION --> APLICADA: aprobar cambios
    EN_REVISION --> DESCARTADA: descartar
    CARGADA --> CON_ERRORES
    PROCESADA --> CON_ERRORES
```

Reglas: una lista no modifica costos ni precios sin revision y aprobacion.

## Item de lista

```mermaid
stateDiagram-v2
    [*] --> NO_RECONOCIDO
    NO_RECONOCIDO --> POSIBLE_COINCIDENCIA
    POSIBLE_COINCIDENCIA --> RECONOCIDO
    NO_RECONOCIDO --> NUEVO_PRODUCTO
    NO_RECONOCIDO --> IGNORADO
    RECONOCIDO --> IGNORADO
```

## Oferta temporal

```mermaid
stateDiagram-v2
    [*] --> PROGRAMADA
    PROGRAMADA --> ACTIVA: llega fecha inicio
    ACTIVA --> VENCIDA: supera fecha fin
    PROGRAMADA --> INACTIVA
    ACTIVA --> INACTIVA
    INACTIVA --> ACTIVA: reactivar si fecha valida
```

## Turno de caja

```mermaid
stateDiagram-v2
    [*] --> ABIERTO
    ABIERTO --> PENDIENTE_CIERRE
    PENDIENTE_CIERRE --> CERRADO
    ABIERTO --> CERRADO: cierre directo
```

## Consumo interno

```mermaid
stateDiagram-v2
    [*] --> REGISTRADO
    REGISTRADO --> PENDIENTE_APROBACION: requiere aprobacion
    PENDIENTE_APROBACION --> APROBADO
    PENDIENTE_APROBACION --> RECHAZADO
    REGISTRADO --> CONFIRMADO: no requiere aprobacion
    APROBADO --> CONFIRMADO
```

Regla: un consumo rechazado no descuenta stock.

## Compra simple

```mermaid
stateDiagram-v2
    [*] --> BORRADOR
    BORRADOR --> RECIBIDA: confirmar recepcion
    BORRADOR --> ANULADA: cancelar
    RECIBIDA --> [*]
    ANULADA --> [*]
```

Regla: `BORRADOR` representa lo esperado y no impacta stock. Al controlar la recepcion se cargan cantidades recibidas reales, vencimientos/lotes opcionales y diferencias. Solo `RECIBIDA` genera movimientos de stock. No representa una factura fiscal ni un pago a proveedor.

## Pedido a proveedor

```mermaid
stateDiagram-v2
    [*] --> BORRADOR
    BORRADOR --> ENVIADO
    BORRADOR --> CANCELADO
    ENVIADO --> RECIBIDO_PARCIAL
    ENVIADO --> RECIBIDO
    ENVIADO --> CANCELADO
    RECIBIDO_PARCIAL --> RECIBIDO
    RECIBIDO_PARCIAL --> CANCELADO
```

## Alerta operativa

```mermaid
stateDiagram-v2
    [*] --> ACTIVA
    ACTIVA --> LEIDA
    LEIDA --> RESUELTA
    ACTIVA --> RESUELTA
    RESUELTA --> [*]
```
