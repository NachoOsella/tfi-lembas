# Flujos criticos

## Venta mostrador

```mermaid
sequenceDiagram
    actor E as Empleado
    participant UI as Angular
    participant V as VentasService
    participant S as StockService
    participant C as CajaService
    participant A as AuditoriaService
    participant DB as PostgreSQL

    E->>UI: Confirma venta
    UI->>V: POST /api/v1/sales/{id}/confirm
    V->>V: Validar venta BORRADOR
    V->>S: Descontar stock FEFO
    S->>DB: Guardar movimientos EGRESO_VENTA
    V->>C: Registrar ingreso por venta
    C->>DB: Guardar MovimientoCaja
    V->>DB: Cambiar venta a CONFIRMADA
    V->>A: Auditar confirmacion
    A->>DB: Guardar auditoria
    V-->>UI: Venta confirmada
```

## Aplicacion de lista de precios

```mermaid
flowchart TD
    A[Seleccionar proveedor] --> B[Cargar Excel, WhatsApp o manual]
    B --> C[Normalizar items]
    C --> D[Reconocer productos]
    D --> E{Estado de item}
    E -->|Reconocido| F[Comparar costo]
    E -->|Dudoso| G[Resolver manualmente]
    E -->|Ignorado| H[No aplicar]
    G --> F
    F --> I[Calcular precio sugerido]
    I --> J[Dueña aprueba seleccion]
    J --> K[Actualizar costo e historial]
    K --> L[Actualizar precio si aprobado]
    L --> M[Marcar etiqueta pendiente]
    M --> N[Auditar]
```

## Consumo interno

```mermaid
flowchart LR
    A[Seleccionar producto] --> B[Indicar cantidad]
    B --> C[Seleccionar razon]
    C --> D{Requiere aprobacion?}
    D -->|No| E[Confirmar]
    D -->|Si| F[Pendiente]
    F --> G[Aprobar]
    G --> E
    E --> H[Descontar FEFO]
    H --> I[Movimiento CONSUMO_INTERNO]
    I --> J[Auditoria]
```

## Cierre de caja

```mermaid
flowchart TD
    A[Turno abierto] --> B[Consultar movimientos]
    B --> C[Calcular saldo esperado]
    C --> D[Ingresar saldo contado]
    D --> E[Calcular diferencia]
    E --> F{Hay diferencia?}
    F -->|Si| G[Registrar observacion]
    F -->|No| H[Confirmar cierre]
    G --> H
    H --> I[Bloquear cierre]
    I --> J[Auditar]
```

## FEFO

```mermaid
flowchart TD
    A[Solicitar egreso de producto] --> B[Obtener lotes con cantidad > 0]
    B --> C[Ordenar por fecha de vencimiento ascendente]
    C --> C2[Dejar lotes sin vencimiento al final]
    C2 --> D[Tomar del lote que vence primero]
    D --> E{Cantidad cubierta?}
    E -->|No| D
    E -->|Si| F[Crear movimientos por lote]
    F --> G[Actualizar cantidades]
```


## Compra simple a proveedor

```mermaid
flowchart TD
    A[Crear compra BORRADOR] --> B[Seleccionar proveedor]
    B --> C[Agregar productos esperados]
    C --> D[Cargar cantidad y costo esperado]
    D --> E[Guardar sin impactar stock]
    E --> F[Llega proveedor]
    F --> G[Abrir control de recepcion]
    G --> H[Cargar cantidad recibida real]
    H --> I[Cargar vencimiento/lote opcional]
    I --> J{Coincide con lo esperado?}
    J -->|Si| K[Marcar OK]
    J -->|No| L[Registrar diferencia]
    K --> M[Confirmar recepcion]
    L --> M
    M --> N[Generar lotes y movimientos INGRESO_COMPRA]
    N --> O[Actualizar costo proveedor opcional]
    O --> P[Auditar]
```

> [!important]
> Regla central: el stock ingresado se calcula con la cantidad recibida real, no con la cantidad esperada.
