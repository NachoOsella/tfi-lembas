# Contratos DTO

## Convenciones

- Usar sufijos `Request` y `Response`.
- No exponer entidades JPA directamente.
- Validar campos obligatorios en backend.
- Fechas en ISO-8601.
- Montos con decimal, no `float` binario.

## ProductRequest

```json
{
  "name": "Avena integral 1kg",
  "barcode": "7790000000012",
  "categoryId": 1,
  "brandId": 2,
  "presentation": "1 kg",
  "salePrice": 3200.00,
  "stockMin": 5,
  "stockIdeal": 20,
  "requiresExpiration": true,
  "usualSupplierId": 3
}
```

## SaleConfirmResponse

```json
{
  "saleId": 15,
  "status": "CONFIRMED",
  "total": 6400.00,
  "cashMovementId": 88,
  "stockMovements": [101, 102]
}
```

## ErrorResponse

```json
{
  "code": "STOCK_NOT_AVAILABLE",
  "message": "No hay stock suficiente para confirmar la venta.",
  "details": [
    {
      "field": "productId",
      "value": 7,
      "reason": "Stock disponible: 1, solicitado: 3"
    }
  ]
}
```

## Estados normalizados

| Concepto | Estados |
|---|---|
| Venta | BORRADOR, CONFIRMADA, ANULADA |
| Lista de precios | CARGADA, PROCESADA, EN_REVISION, APLICADA, DESCARTADA, CON_ERRORES |
| Item de lista | RECONOCIDO, POSIBLE_COINCIDENCIA, NO_RECONOCIDO, NUEVO_PRODUCTO, IGNORADO |
| Turno caja | ABIERTO, PENDIENTE_CIERRE, CERRADO |
| Compra simple | BORRADOR, RECIBIDA, ANULADA |
| Detalle compra | PENDIENTE, RECIBIDO_OK, RECIBIDO_CON_DIFERENCIA, NO_RECIBIDO |
| Consumo interno | REGISTRADO, PENDIENTE_APROBACION, APROBADO, RECHAZADO, CONFIRMADO |
