# Casos de prueba

| ID | Caso | Resultado esperado |
|---|---|---|
| CP-01 | Login con credenciales validas. | Acceso concedido con rol correcto. |
| CP-02 | Login con usuario inactivo. | Acceso rechazado. |
| CP-03 | Empleado accede a endpoint admin. | Respuesta 403. |
| CP-04 | Crear producto con codigo de barras unico. | Producto creado. |
| CP-05 | Crear producto con codigo duplicado activo. | Error de validacion. |
| CP-06 | Registrar ingreso con vencimiento. | Lote y movimiento creados. |
| CP-07 | Descontar stock FEFO. | Sale primero el lote con vencimiento mas cercano. |
| CP-08 | Intentar egreso mayor al stock. | Operacion rechazada salvo permiso. |
| CP-09 | Confirmar venta valida. | Venta confirmada, stock descontado, caja creada. |
| CP-10 | Confirmar venta sin stock. | No confirma, no genera caja. |
| CP-11 | Anular venta confirmada. | Estado/movimientos compensatorios y auditoria. |
| CP-12 | Modificar precio sin permiso. | Operacion rechazada. |
| CP-13 | Aplicar oferta vigente. | Detalle guarda oferta y precio aplicado. |
| CP-14 | Registrar egreso de caja sin concepto. | Error de validacion. |
| CP-15 | Cerrar caja con diferencia. | Diferencia y observacion persistidas. |
| CP-16 | Registrar consumo interno. | Descuenta stock y no genera caja. |
| CP-17 | Registrar merma con motivo. | Movimiento MERMA creado. |
| CP-18 | Cargar lista Excel valida. | Lista e items quedan en revision. |
| CP-19 | Cargar texto WhatsApp ambiguo. | Items dudosos quedan para revision. |
| CP-20 | Descartar lista. | No cambia costos ni precios. |
| CP-21 | Aplicar costo y precio aprobado. | Historiales creados y etiqueta pendiente. |
| CP-22 | Calcular precio sugerido. | Aplica margen y redondeo configurados. |
| CP-23 | Reporte stock bajo. | Lista productos bajo minimo. |
| CP-24 | Alerta duplicada. | No se crea duplicado si la condicion no cambio. |
| CP-25 | Validacion DTO invalido. | Respuesta 400 con detalles. |
| CP-26 | OpenAPI disponible. | Documentacion accesible en ambiente dev/demo. |
| CP-27 | Confirmar compra simple con productos validos. | Compra queda recibida y genera movimientos INGRESO_COMPRA. |
| CP-28 | Confirmar compra con producto que controla vencimiento sin fecha. | El sistema advierte que no generara alertas y permite confirmar si la configuracion lo permite. |
| CP-29 | Confirmar compra con cantidad esperada distinta a recibida. | Ingresa stock por cantidad recibida y registra diferencia. |
