# Estrategia de testing

## Objetivo

Validar reglas criticas del dominio, consistencia transaccional, seguridad por roles y flujos principales de usuario.

## Niveles

| Nivel | Objetivo | Ejemplos |
|---|---|---|
| Unitario | Probar reglas aisladas. | FEFO, calculo de precio sugerido, validacion de oferta. |
| Integracion | Probar servicios con base de datos. | Confirmar venta, aplicar lista, cerrar caja. |
| API | Verificar contratos REST. | Login, crear producto, confirmar venta. |
| Frontend | Validar componentes y flujos UI. | Nueva venta, busqueda de producto, errores. |
| End-to-end | Demostrar circuito completo. | Crear producto, cargar stock, vender, ver caja. |

## Prioridades de prueba

1. Confirmacion de venta transaccional.
2. Descuento FEFO.
3. Stock insuficiente.
4. Modificacion de precio con permiso y motivo.
5. Caja generada por venta.
6. Consumo interno sin caja.
7. Aplicacion de lista de precios con aprobacion.
8. Etiquetas pendientes.
9. Compra simple confirmada con ingreso de stock.
10. Seguridad por roles.
10. Auditoria.

## Criterios de calidad

- No debe haber procesos criticos sin prueba.
- Las validaciones definitivas deben estar en backend.
- Los errores deben ser predecibles y documentados.
- La documentacion debe estar alineada con requisitos y pruebas.
