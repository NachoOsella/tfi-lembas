# Frontend

## Tecnologia

Angular para construir una SPA responsive, mobile-first y orientada a uso operativo en mostrador.

## Features sugeridas

```text
app/
  core/
  shared/
  auth/
  dashboard-admin/
  dashboard-employee/
  products/
  stock/
  sales/
  cash/
  suppliers/
  purchases/
  price-lists/
  pricing/
  offers/
  consumption/
  labels/
  reports/
  audit/
```

## Principios UX

- Mobile-first.
- Botones grandes para uso tactil.
- Campo de scanner con foco automatico en venta.
- Busqueda rapida de productos.
- Diferenciacion clara entre home admin y home empleado.
- Formularios cortos para tareas de mostrador.
- Confirmaciones antes de acciones irreversibles.
- Errores claros y accionables.

## Estado y seguridad

- Guards por rol para rutas.
- Interceptor HTTP para token o sesion.
- Manejo centralizado de errores.
- No confiar en ocultamiento visual como control de seguridad: backend valida permisos.

## Pantallas clave

| Pantalla | Usuario principal | Objetivo |
|---|---|---|
| Login | Todos | Acceso seguro. |
| Home admin | Administradora | Indicadores, alertas y accesos de gestion. |
| Home empleado | Empleado | Nueva venta, tareas y cierre de turno. |
| Nueva venta | Empleado | Escanear, buscar, cobrar internamente y confirmar. |
| Productos | Administradora | Gestion de catalogo. |
| Stock | Administradora/empleado autorizado | Lotes, vencimientos y movimientos. |
| Compras simples | Administradora/Autorizado | Registrar compras minimas con ingreso de stock. |
| Listas de precios | Administradora | Cargar, revisar y aplicar cambios. |
| Caja | Empleado/admin | Movimientos y cierres. |
| Reportes | Administradora | Informacion operativa. |
