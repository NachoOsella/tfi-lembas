# Seguridad

## Objetivo

Proteger informacion sensible del negocio, asegurar que cada usuario opere segun su rol y registrar acciones criticas.

## Controles principales

| Control | Descripcion |
|---|---|
| Autenticacion | Login con credenciales y token JWT o sesion segura. |
| Hash de contraseña | Contraseñas almacenadas con algoritmo seguro. |
| Autorizacion backend | Validacion de permisos en endpoints y servicios. |
| Guards frontend | Mejora de UX, no reemplaza controles backend. |
| Baja logica de usuarios | Usuarios inactivos no pueden operar. |
| Auditoria | Acciones criticas con usuario, fecha, entidad y detalle. |
| CORS restringido | Origenes permitidos segun ambiente. |
| Headers de seguridad | Configurados desde backend/Nginx. |

## Informacion sensible

No debe mostrarse al empleado salvo permiso explicito:

- costos de proveedor;
- margenes;
- reportes financieros globales;
- auditoria completa;
- configuracion global;
- gestion de usuarios;
- importaciones masivas y aplicacion de precios.

## Permisos sugeridos

| Permiso | Uso |
|---|---|
| PRODUCT_WRITE | Crear o modificar productos. |
| STOCK_ADJUST | Ajustar stock manualmente. |
| SALE_PRICE_OVERRIDE | Modificar precio en venta. |
| SALE_CANCEL | Anular venta. |
| CASH_CLOSE_DAILY | Cierre diario de caja. |
| SUPPLIER_MANAGE | Gestionar proveedores. |
| PRICE_LIST_APPLY | Aplicar listas de precios. |
| REPORT_VIEW_FINANCIAL | Ver reportes sensibles. |
| AUDIT_VIEW | Consultar auditoria. |

## Riesgos y mitigaciones

| Riesgo | Mitigacion |
|---|---|
| Empleado ve costos | Control de permisos en API y UI. |
| Cambio de precio no autorizado | Permiso, motivo obligatorio y auditoria. |
| Venta sin caja consistente | Transaccion backend. |
| Manipulacion de auditoria | Auditoria no editable por usuarios comunes. |
| Token expuesto | Expiracion, HTTPS y almacenamiento prudente. |
