# Backend

## Tecnologia

Spring Boot con Spring Web, Spring Data JPA, Spring Security, Bean Validation, PostgreSQL, migraciones Flyway o Liquibase y documentacion OpenAPI/Swagger.

## Estructura sugerida

```text
com.lembas
  security
  users
  products
  stock
  sales
  cash
  suppliers
  purchases
  pricelists
  pricing
  offers
  consumption
  labels
  reports
  alerts
  audit
  shared
```

Cada modulo puede organizarse internamente en:

```text
api          Controllers REST y DTOs
action       Casos de uso / servicios de aplicacion
domain       Entidades, reglas e interfaces
infra        Repositorios, adaptadores y persistencia
```

## Responsabilidades

- Exponer API REST versionada.
- Validar entradas con DTOs.
- Ejecutar reglas de negocio criticas.
- Coordinar transacciones.
- Persistir entidades y movimientos.
- Registrar auditoria.
- Documentar endpoints con OpenAPI.

## Transacciones criticas

| Proceso | Componentes | Criterio |
|---|---|---|
| Confirmar venta | sales, stock, cash, audit | Todo se confirma o todo se revierte. |
| Consumo interno | consumption, stock, audit | Descuenta stock sin caja. |
| Aplicar lista | pricelists, pricing, labels, audit | Requiere revision y aprobacion. |
| Ajuste de stock | stock, audit | Requiere motivo y movimiento. |
| Confirmar recepcion de compra simple | purchases, stock, suppliers, audit | Usa cantidades recibidas reales, genera lotes/movimientos de stock y actualiza costo proveedor opcionalmente. |
| Cierre de caja | cash, audit | Bloquea cierre confirmado. |

## Manejo de errores

- `BusinessException` para reglas de negocio.
- `NotFoundException` para recursos inexistentes.
- `ForbiddenException` para permisos insuficientes.
- Respuestas con codigo, mensaje y detalles validables.
- No exponer stack traces al frontend.

## Buenas practicas

- Controllers delgados.
- Servicios orientados a casos de uso.
- DTOs de request/response.
- Repositorios por agregado o entidad principal.
- Pruebas unitarias para reglas y pruebas de integracion para transacciones.
